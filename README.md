# Chognari Dogs — каталог собак (этап 1: сайт)

Волонтёрский проект помощи собакам: город **Самтредиа** и село **Чогнари** (Грузия).
Сайт — это рабочий каталог-инструмент: ссылку дают клиентам через Facebook/WhatsApp,
люди смотрят собак и **звонят по телефону** (форм нет).

## Что уже сделано (этап «сайт»)

- Публичная часть на **Astro** (чистый HTML/CSS/JS), готова к хостингу на **Cloudflare Pages**.
- Структура: **Собаки · Условия · Помочь** (вкладки).
- Каталог с фильтром: **Пол + стерилизация** (Мальчик / Девочка стерильная / Девочка нестерилизованная),
  **Вес** (диапазоны кг), **Возраст**, **Характер**. Логика: внутри группы ИЛИ, между группами И.
- Карточка: главное фото + **смена на второе фото при наведении**; по клику — **«паспорт»** (попап)
  с галереей (3–5 фото), полным текстом и параметрами.
- **Двуязычие**: грузинский крупно + русский мелким шрифтом (для Андрея).
- Крупный **телефон и имя куратора** в шапке, на мобильном — закреплённая кнопка «Позвонить».
- В шапке — **два герба** (Самтредиа + знак села) — сейчас это ЗАГЛУШКИ, см. ниже.

## Как посмотреть прямо сейчас (без установки)

Откройте файл **`preview.html`** двойным кликом в браузере — это автономная версия
всего сайта (можно показывать заказчику, пересылать).

## Как запустить рабочую версию (для разработки)

```bash
npm install
npm run dev      # локальный просмотр на http://localhost:4321
npm run build    # сборка в dist/
```

## Деплой на Cloudflare Pages (бесплатно)

1. Залить репозиторий на GitHub.
2. В Cloudflare → Pages → Create → Connect to Git.
3. Build command: `npm run build`, Output directory: `dist`.
4. Каталог `functions/` подхватится автоматически (там будет API для админки на этапе 2–3).

## Где что лежит

```
src/data/dogs.json   — карточки собак (ВРЕМЕННО; на этапе 3 заменим на /api/dogs из базы)
src/data/site.json   — все тексты интерфейса (грузинский + русский)
src/components/       — шапка, фильтр-блоки, паспорт, подвал
src/pages/index.astro — страница + клиентская логика (фильтр, паспорт, галерея)
public/gerb-*.svg     — ЗАГЛУШКИ гербов
functions/            — место для API (этап 2–3: админка, база, фото)
```

## ВАЖНО — что нужно заменить/проверить

- **Грузинский текст** в `site.json` и `dogs.json` сделан вчерне и **должен проверить носитель**
  (та самая женщина-куратор). Русский — для Андрея, мелким шрифтом.
- **Гербы**: заменить `public/gerb-samtredia.svg` и `public/gerb-chognari.svg` на реальные
  (официальный герб Самтредиа + знак села с обложки группы). Можно PNG/SVG, формат тот же.
- **Имя и телефон куратора** — в `site.json` → `contact` (сейчас стоит пример Натия / +995 597 934 007).
- **Фото собак** сейчас заглушки с placedog.net — заменим реальными (через админку на этапе 2).

## Решения по вопросам заказчика (отражены в коде)

- Резервирования нет. «Авто-скрытие пристроенных» сделано: у собаки поле `status`;
  если не `searching` — она не показывается (на этапе 2 это будет переключатель в админке).
- Удаление собаки сетку **не ломает** — карточки просто перестраиваются, пустых ячеек не остаётся.
- Блок «Уже нашли дом» не делали (Андрей сказал — скорее не нужен). Легко добавить позже.
- Реквизитов/донатов нет — раздача бесплатная; в «Помочь» только реальные способы (взять, рассказать, корм, позвонить).

## Этап 2 — Админка (кликабельная версия)

Адрес: **`/admin`** (на live-сайте — `https://…pages.dev/admin`).
Быстрый просмотр без сервера — открыть **`admin-preview.html`** двойным кликом.

**Вход:** демо-пароль `chognari`.

Что умеет:
- Список собак: миниатюра, кличка, статус, параметры. Поиск по кличке, фильтр по статусу (все / ищут дом / пристроены).
- Добавить / редактировать карточку: кличка, **фото 1–5** (перетаскивание или выбор, выбрать главную, поменять порядок, удалить),
  пол + стерилизация, вес в кг (диапазон для фильтра считается автоматически), возраст, характер (мультивыбор),
  ладит с курами, порода (груз./рус.), локация, описание (груз./рус.), статус.
- Быстрое переключение статуса в один клик. «Пристроена» = собака скрыта в каталоге на сайте.
- Удаление с подтверждением. Список после удаления **сдвигается**, пустых ячеек не остаётся.
- Изменение порядка вывода (кнопки ▲ ▼).
- Валидация обязательных полей (кличка, фото, вес).

**Важно про этап 2:** данные сохранялись **в браузере** (localStorage). На этапе 3 это
заменено на общую базу — см. ниже.

## Этап 3 — общая база (D1 + R2 + API) ✅ готово

Теперь админка и сайт работают через настоящий бэкенд:

- **D1** — база собак (вместо `dogs.json`). Публичный каталог грузит `GET /api/dogs`.
- **R2** — хранилище фото. Куратор загружает фото в админке, они сжимаются в браузере
  и уходят в R2; на сайте отдаются по ссылкам `/img/...`.
- **Pages Functions** (`functions/`) — API: CRUD собак, загрузка фото, вход.
- **Вход** — пароль в переменной окружения Cloudflare (`ADMIN_PASSWORD`), сессия в
  подписанной HttpOnly-cookie. Пароля в коде нет.

Правки из админки сразу видны на сайте и на любом устройстве.

### Что нужно сделать один раз (настройка Cloudflare)

Всё выполняется в папке проекта. Нужен установленный Node.js. Команды можно копировать.

```bash
# 1. Установить зависимости
npm install

# 2. Войти в свой аккаунт Cloudflare (откроется браузер)
npx wrangler login

# 3. Создать базу данных D1
npx wrangler d1 create chognari-dogs
#    -> команда напечатает database_id. Скопируй его в wrangler.toml
#       (строка database_id = "ВСТАВЬ_СЮДА_database_id")

# 4. Создать таблицу и залить 6 стартовых собак
npx wrangler d1 execute chognari-dogs --remote --file schema.sql

# 5. Создать R2-бакет для фото (требует включённого R2 в дашборде)
npx wrangler r2 bucket create chognari-photos

# 6. Собрать сайт
npm run build

# 7. Опубликовать (первый раз спросит имя проекта — можно chognari-dogs)
npx wrangler pages deploy dist

# 8. Задать пароль админки (введёшь значение в ответ на запрос)
npx wrangler pages secret put ADMIN_PASSWORD
```

После этого: сайт — на `https://<проект>.pages.dev`, админка — `/admin`, вход по паролю
из шага 8. Для повторной публикации после правок кода — снова `npm run build` и
`npx wrangler pages deploy dist`.

> R2 при первом включении просит подтвердить аккаунт (привязать карту). В рамках
> бесплатного лимита (10 ГБ) деньги не списываются.

### Локальная разработка с базой

```bash
cp .dev.vars.example .dev.vars          # впиши пароль внутрь .dev.vars
npx wrangler d1 execute chognari-dogs --local --file schema.sql   # локальная копия базы
npm run build
npx wrangler pages dev dist             # http://localhost:8788, с D1/R2/паролем
```
